Install¶

What we use...¶

...for the simulation¶

The program is written in python 2.7. It should work on python 3.2. For some tutorial, see dive into python.

It requires numpy. For Matlab users, see NumPy for Matlab users.

Numpy supports n-dimensionnal arrays (class numpy.ndarray) and matrices (built with numpy.matrix) which are 2-d matrices with special behavior, similar to the matlab one. In arboris-python we never uses matrices, they are evil.

We also use (they are useful but not mandatory):

  • h5py to store simulation results in the HDF5 format,
  • matplotlib to plot results, like Matlab.

...for the visualization¶

We use 2 different ways to visualize simulations:

  • Vpython which is a python module which can easily be connected with arboris-python.
  • Daenim, which is a program developed in parallel with Arboris-python, dedicated to the visualization of collada files. It can be plugged on Arboris-python as an observer in order to replace Vpython. It allows to display the scene with more complex shapes (by mixing collada files) and to save snapshots to create movies.

...for the documentation¶

The documentation is written in the reST markup language and processed by sphinx (version >=6). See the sphinx (and reST) documentation online. We use sphinx plugins which allow

  • to embed latex in the doc,
  • to parse the docstrings spread in the code,
  • to run the examples with doctest,
  • to generate the uml diagrams using graphviz.

Installation on Ubuntu¶

Installing arboris-python dependancies¶

sudo apt-get install  ipython  python-numpy  python-h5py  python-matplotlib python-visual

Installing arboris-python¶

When you have downloaded Arboris-python, go in the new directory, open a terminal and run:

sudo python setup.py install

if you have no administration rights, you can install locally:

python setup.py install --user

Installing the tools needed to build the documentation¶

sudo aptitude install python-sphinx graphviz texlive-latex-base dvipng

For Mac OS 10.6 (Snow Leopard)¶

Installing dependancies¶

...manually¶

If you choose to install dependancies manually, be aware that VPython requires a 32bit version of python. See this page for more info. Else, do not install VPython (it is not mandatory).

Install...

  • python 2.7 (or if you want to try, python 3.2),
  • numpy; Ensure to choose the same python version,
  • h5py,
  • matplotlib,
  • VPython.

...from Macports¶

Mac OS ships with python 2.5, 2.6 and numpy pre-installed. The other packages can be easily installed using macports:

sudo port install py27-ipython  py27-numpy  py27-h5py  py27-matplotlib

Installing arboris-python¶

When you have downloaded Arboris-python, go in the new directory, open a terminal and run:

sudo python setup.py install

if you have no administration rights, you can install locally:

python setup.py install --user

For windows¶

Installing dependancies¶

...manually¶

Install...

  • python 2.7, or if you want to try, python 3.2,
  • numpy; Ensure to choose the same python version,
  • h5py,
  • matplotlib,
  • VPython.

For windows 64bits, these modules are available in this unofficial website.

...from Python(x,y)¶

All these programs (and many others) are conveniently packaged by the Python(x,y) project, you may install them from there.

Installing arboris-python¶

When you have downloaded Arboris-python, open a command prompt in the arboris-python folder (shift + right click on windows 7) and run:

setup.py install
